      📚 目录

      新搭建一个前端项目的时候，必须要做的事情

      1. 选择要使用的基础框架
      2. 选择状态/数据流管理工具
      3. 添加好开源组件库/或自己封装的组件库
      4. 选择style的预编译语言/或使用style in js
      5. 在index.less 中处理好标签的默认样式
      6. 定义字体样式
      7. 添加在不同窗口大小下的 font-size
      8. 定义好color
      @text-primary-color: #ddd;
@text-second-color: #eee;
@divider: 1px solid #ddd;
@primary-color: blue;

// ... and more
      1. 定义好常用的mixin方法
      1. 使用css 变量或其他方式，预留主题设置功能
      2. 添加好ts配置/在使用cli时选择对应的
      3. 配置好jest/mocha 可以进行前端测试
      4. 加好文档、写明架构，文件结构
      5. 加好代码检查，eslint、style-lint 等、并在文档中写明使用的规范，包括tsconfig
      6. 配置好webpack, 使用compression 打包压缩后的文件 加好webpack-analysis的脚本、随时可查看
      7. 做好代码分割、在文档中写明
      8. 文档中写明使用clean的code、并在review时注意
      9. http请求封装、auth、权限处理好 http可正确抛出异常
      10. constant 单独定义，并在文档中写明。如 api path、reducer type等
      11. 要求提取公共方法，封装公共组件，提供使用文档，和通过单元测试
      12. 封装全局组件、confirm、alert、empty、noSearch、loading等
      13. 封装所需的高阶组件
      14. ci没配的话先加一个部署脚本
      15. 配置好代理/mock地址
      16. 确认接口规范
      17. 封装好storage
      18. 封装好表单组件/table组件
      19. 添加代码提交前的eslint / stylelint 检查
      20. 关注内存使用
      21. 页面布局处理好 使用响应式还是内部最小宽高滚动
      22. 处理loading状态管理
      23. form
      24. … and more
